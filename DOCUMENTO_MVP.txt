================================================================================
1. RESUMEN EJECUTIVO
================================================================================

Sistema CMS web que permite a una organizacion gestionar y publicar contenido
(noticias, testimonios, solicitudes de contacto, comentarios, archivos) para
multiples paises desde un solo panel de administracion con control de acceso
por roles. Incluye bitacora de auditoria, versionado de contenido,
notificaciones, y portales publicos informativos por pais.

================================================================================
2. OBJETIVOS DEL MVP
================================================================================

OV-01  Proveer portales publicos de informacion por pais (noticias + testimonios)
OV-02  Permitir la gestion centralizada de contenido desde un panel
       administrativo con control de acceso por roles
OV-03  Recibir y gestionar solicitudes de contacto del publico
OV-04  Implementar autenticacion segura con recuperacion de contrasena
OV-05  Desplegar el sistema en produccion mediante Docker + CI/CD

5. ROLES DE USUARIO
================================================================================

Superadmin
  Acceso completo a todos los paises. Puede gestionar usuarios y todo el
  contenido.

Admin Pais
  Gestiona contenido (noticias, testimonios, solicitudes) unicamente de su
  pais asignado. No puede crear/editar usuarios.

Editor
  Crea y edita noticias y testimonios de su pais. No puede eliminar ni
  gestionar solicitudes de contacto.

Visitante
  Acceso solo a portales publicos. No requiere autenticacion.

6. ARQUITECTURA TECNICA
================================================================================

Cliente (Navegador)
  HTML + CSS + JS vanilla -> Puerto 3001
       |
       v
Express API (backend/)
  Routes -> Controllers -> Services -> Repositories
  Middlewares: auth, roles, validation, error handler, rate limiter
  Documentacion: Swagger UI en /api/docs
       |
       v
Supabase (PostgreSQL)
  Tablas: paises, roles, usuarios, noticias, testimonios,
          solicitudes_contacto, refresh_tokens, archivos,
          auditoria, categorias, noticia_categorias,
          estadisticas_pais, notificaciones,
          historial_noticias, comentarios,
          historial_conexiones

Stack tecnico:
  Backend:   Node.js v20, Express 5
  Base de datos: PostgreSQL (Supabase)
  Frontend:  HTML5, CSS3, JavaScript vanilla
  Autenticacion: JWT + bcryptjs con rotacion de refresh tokens
  Documentacion API: Swagger (swagger-jsdoc + swagger-ui-express)
  Rate limiting: express-rate-limit
  Tests:     Jest + Supertest
  Infraestructura: Docker, docker-compose, Oracle Cloud
  CI/CD:     GitHub Actions

8. FLUJO DE USUARIO - MVP
================================================================================

Visitante:
  Landing -> Explora noticias destacadas -> Accede a portal de pais
          -> Lee noticias / testimonios -> Envia contacto o testimonio

Administrador:
  Login -> Dashboard -> Gestiona noticias / testimonios / solicitudes
         -> Cierra sesion

Superadmin:
  Login -> Dashboard -> Gestiona contenido (todos los paises)
         -> Gestiona usuarios -> Cierra sesion
